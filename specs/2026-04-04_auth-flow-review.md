# 認証フロー仕様書 — レビュー指摘詳細

## レビュー実施日: 2026-04-04

## レビュアー別指摘一覧

### A: セキュリティ・プライバシー

| # | 重要度 | 指摘 | 対応 |
|---|--------|------|------|
| S-1 | HIGH | callbackUrl のオープンリダイレクト脆弱性 | REQ-7 に同一オリジン制限を明記 |
| S-2 | HIGH | JWT 署名鍵（AUTH_SECRET）の仕様欠如 | JWT セキュリティ要件テーブル追加 |
| S-3 | HIGH | Cookie 属性の仕様未定義 | JWT セキュリティ要件テーブル追加 |
| S-4 | HIGH | ADMIN_EMAILS のソースコードハードコード | ADR-002 で方針を明文化。現時点は許容 |
| S-5 | MEDIUM | OAuth PKCE/state パラメータの仕様欠如 | OAuth セキュリティセクション追加 |
| S-6 | MEDIUM | セッションハイジャック対策の仕様欠如 | JWT callback で毎回 role 再計算している旨を REQ-5 に注記 |
| S-7 | MEDIUM | CSRF 対策の仕様欠如 | CSRF 対策セクション追加 |
| S-8 | MEDIUM | エラー情報の漏洩リスク | T-19 テストケース追加 |
| S-9 | MEDIUM | JWT 内 email の暗号化 | JWE（NextAuth デフォルト）を JWT セキュリティ要件に明記 |
| S-10 | LOW | レートリミットの仕様欠如 | 認証フロー仕様の範囲外として保留 |
| S-11 | LOW | trustHost: true の安全性 | JWT セキュリティ要件に根拠を記載 |

### B: DB/API 技術整合性

| # | 重要度 | 指摘 | 対応 |
|---|--------|------|------|
| D-1 | HIGH | users.role と JWT role の二重管理 | TD-1 として記録 |
| D-2 | HIGH | API 認証パターンの不統一（2ルート） | TD-2 として記録 |
| D-3 | MEDIUM | sessions テーブルの残存 | DB テーブルセクションに「将来 DROP 予定」と注記 |
| D-4 | MEDIUM | D1 + DrizzleAdapter 互換性の ADR 未作成 | ADR-001 追加 |
| D-5 | MEDIUM | getAuthHandler の毎回初期化 | TD-5 として記録 |
| D-6 | LOW | DrizzleAdapter の as any キャスト | TD-3 に含む |
| D-7 | LOW | JWT 構造の iat/exp 記載が不正確 | 「NextAuth 自動管理」と注記追加 |
| D-8 | LOW | verification_tokens テーブル残存 | DB テーブルセクションに注記 |

### C: UX・フロー・エッジケース

| # | 重要度 | 指摘 | 対応 |
|---|--------|------|------|
| U-1 | CRITICAL | callbackUrl 検証なし（オープンリダイレクト） | REQ-7 に追記（S-1 と重複） |
| U-2 | CRITICAL | セッション期限切れ時の UX 未定義 | REQ-4 に NextAuth の自動処理を明記 |
| U-3 | HIGH | 複数タブでのログイン/ログアウト同期なし | REQ-8 として追加 |
| U-4 | HIGH | callbackUrl の長さ・形式の上限未定義 | NextAuth のデフォルト制限に委ねる（仕様範囲外） |
| U-5 | HIGH | 非管理者の /admin ログイン失敗 UX が不親切 | REQ-6 にエラーメッセージ要件追加 |
| U-6 | HIGH | ログインページの i18n 対応欠落 | TD-6 として記録 |
| U-7 | MEDIUM | ロケール切り替え時のセッション維持が未記載 | T-11 テストケース追加 |

| U-10 | MEDIUM | モバイルブラウザでの OAuth リダイレクト考慮不足 | PWA 対応時に検討。現時点は範囲外 |
| U-11 | MEDIUM | /checkout/complete 除外のクエリパラメータ考慮 | パスのみで判定、問題なし |
| U-12 | LOW | 管理者ログインページの英語ハードコード | TD-6 に含む |
| U-13 | LOW | signOut リダイレクト先のロケール記載が簡略 | フロー図に /[locale] と記載済み |
| U-14 | LOW | sessions/verification_tokens の未使用テーブル | D-3, D-8 と重複 |
| U-15 | LOW | ログイン済みで /auth/login にアクセスした場合が未定義 | T-12 テストケース追加 |

| R-2 | HIGH | callbackUrl のバリデーション（S-1, U-1 と重複） | REQ-7 に追記 |
| R-3 | HIGH | 管理者ログイン失敗 UX（U-5 と重複） | REQ-6 に追記 |
| R-4 | HIGH | 一般ユーザー向け API の認証要件未定義 | 認証チェックポイント一覧に追加 |
| R-5 | MEDIUM | JWT 期限切れ時の具体的挙動が曖昧 | REQ-4 に補足 |
| R-6 | MEDIUM | ログインページの i18n（U-6 と重複） | TD-6 として記録 |
