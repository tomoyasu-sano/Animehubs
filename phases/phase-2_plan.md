# Phase 2: カート + 予約フロー + メール送信

## Phase概要
カート機能（ローカルストレージ + サイドパネル）、予約フロー（チェックアウトフォーム、在庫確認、予約作成）、Resendによるメール送信を実装する。このPhase完了後、ユーザーが商品を予約できる状態になる。

## 実装範囲

### 1. カート機能
- useCart フック（ローカルストレージベース）
- CartSidebar（スライドインパネル）
- CartItem、CartSummary コンポーネント
- カートページ（/cart）
- ヘッダーのカートアイコンにバッジ表示

### 2. 予約フロー
- CheckoutForm コンポーネント（名前、メール、場所、時間帯）
- LocationPicker コンポーネント
- TimeSlotPicker コンポーネント
- Instagram連絡オプション（場所・時間帯スキップ可）
- チェックアウトページ（/checkout）
- 予約確認完了ページ（/checkout/confirm）

### 3. 予約API
- POST /api/reservations — 予約作成（在庫確認 + トランザクション）
- 在庫不足時のエラーハンドリング

### 4. メール送信
- Resend API 統合
- React Email テンプレート（予約確認メール）
- 多言語対応（ユーザーの閲覧言語で送信）

### 5. お気に入り機能
- useFavorites フック（ローカルストレージ）
- FavoriteButton コンポーネント（商品カード・詳細に配置）

## 単体テスト（TDD）
- useCart: 追加、削除、数量変更、合計計算、クリア
- useFavorites: 追加、削除、トグル
- 予約バリデーション: 名前・メールの形式チェック
- 予約API: 正常系（予約作成 + 在庫減算）、在庫不足エラー
- メール送信: テンプレート生成（モック使用）

## Phase完了時テスト
- **機能的E2E**: 商品詳細 → カート追加 → カート確認 → チェックアウト → 予約確定 → 確認ページ
- **視覚的回帰**: カートサイドパネル、チェックアウトページ、確認ページ
- **パフォーマンス**: チェックアウトページ
