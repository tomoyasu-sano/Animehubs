# Security Review Command

セキュリティ脆弱性の検出と修正提案を行う。@security-reviewer エージェントを起動してレビューを実行する。

## Usage

`/security-review [path|.]`

## 実行フロー

1. 対象ファイルのセキュリティレビューを実行
2. レビュー結果をサマリーとして返却

## 対象ファイルの決定

- パス指定あり → そのパス配下を対象
- パス指定なし → `git diff --name-only HEAD` で変更ファイルを対象

## レビュー内容

### OWASP Top 10 チェック
1. インジェクション（SQL, コマンド, XSS）
2. 認証・認可の不備
3. 機密データの露出
4. XXE
5. アクセス制御の不備
6. セキュリティ設定ミス
7. XSS
8. 安全でないデシリアライゼーション
9. 既知の脆弱性を持つコンポーネント
10. 不十分なログ・監視

### コードパターンチェック
- ハードコードされた秘密情報
- ユーザー入力を使ったシェルコマンド
- 文字列結合による SQL
- innerHTML への未サニタイズ入力
- 認証チェックのないルート
- レート制限の欠如

### 依存関係チェック
- `npm audit` / `pip audit` 等で既知の脆弱性を確認

## 出力フォーマット

```
SECURITY REVIEW: [PASS/FAIL]

CRITICAL: X件
HIGH:     X件
MEDIUM:   X件
LOW:      X件

[各指摘の詳細: ファイル:行番号、問題の説明、修正提案]
```

## 修正ループ（サブエージェント内）

app-builder から委任された場合:
- CRITICAL/HIGH が検出された場合、サブエージェント内で修正を試みる
- 最大3回の修正→再レビューループ
- 解消しなければ詳細を app-builder に返却

## Arguments

$ARGUMENTS:
- `[path|.]` - レビュー対象パス（省略時は変更ファイル）
